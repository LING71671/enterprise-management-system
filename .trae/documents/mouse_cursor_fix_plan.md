# 鼠标跟随效果修复计划

## 问题分析

**问题描述**：每次切换管理模块时，鼠标跟随的光点都会移到画面正中央，和鼠标实际位置不重合。

**根本原因**：
- 在 `src/assets/js/main.js` 文件的 `initCustomCursor` 函数中，初始鼠标位置被硬编码为屏幕中心：
  ```javascript
  let mouseX = window.innerWidth / 2;
  let mouseY = window.innerHeight / 2;
  let cursorX = mouseX;
  let cursorY = mouseY;
  ```
- 当页面加载时，无论鼠标当前在什么位置，鼠标跟随效果都会从屏幕中心开始移动。
- 切换管理模块时，页面会重新加载，导致 `initCustomCursor` 函数再次执行，鼠标跟随效果又会回到屏幕中心。

## 解决方案

**目标**：初始化鼠标跟随效果时，使用鼠标的实际位置作为初始位置，而不是屏幕中心。

**实现步骤**：

1. **修改 `initCustomCursor` 函数**：
   - 移除硬编码的屏幕中心初始位置
   - 添加获取当前鼠标实际位置的逻辑
   - 确保在鼠标未移动时也能获取到正确的初始位置

2. **具体修改**：
   - 在 `initCustomCursor` 函数中，使用 `document.addEventListener('mousemove', ...)` 来获取鼠标的实际位置
   - 或者使用 `PointerEvent` API 来获取当前鼠标位置
   - 确保初始位置设置正确，避免页面加载时鼠标跟随效果跳转到屏幕中心

## 技术实现

**修改文件**：`src/assets/js/main.js`

**修改内容**：
1. 移除硬编码的屏幕中心初始位置
2. 添加获取当前鼠标实际位置的逻辑
3. 确保鼠标跟随效果从鼠标实际位置开始，而不是屏幕中心

## 预期效果

- 页面加载时，鼠标跟随的光点会直接出现在鼠标当前位置，而不是屏幕中心
- 切换管理模块时，鼠标跟随的光点会保持在鼠标当前位置，不会跳转到屏幕中心
- 鼠标移动时，鼠标跟随效果依然平滑流畅

## 风险评估

**低风险**：
- 修改仅涉及鼠标跟随效果的初始位置设置，不影响其他功能
- 修改范围小，只需要修改一个函数
- 不会破坏现有功能，只是优化用户体验

**注意事项**：
- 确保在不同浏览器中都能正确获取鼠标位置
- 确保在鼠标未移动时也能获取到正确的初始位置
- 确保修改后的鼠标跟随效果依然平滑流畅

## 测试计划

1. **功能测试**：
   - 打开控制面板，检查鼠标跟随效果是否从鼠标实际位置开始
   - 切换管理模块，检查鼠标跟随效果是否保持在鼠标当前位置
   - 移动鼠标，检查鼠标跟随效果是否平滑流畅

2. **浏览器兼容性测试**：
   - 在不同浏览器中测试鼠标跟随效果
   - 确保在所有浏览器中都能正确获取鼠标位置

3. **响应式测试**：
   - 确保在不同屏幕尺寸下鼠标跟随效果都能正常工作
   - 确保在移动设备上鼠标跟随效果被正确禁用
